0311 수업 정리

1. git이 만들어진 이유 ( 뭐길래 협업이 좋고 전세계적으로 사용하는 툴이 되었는가?)
- 분산 버전 관리 시스템 : 중앙서버에 접속하지 않아도 코드작업이 가능하다.

- 원격저장 시스템은 서버에 작업물을 저장하고 로컬 저장 PC에도 내 작업물을 저장한다.
  허나 작업물을 바로바로 올려야만 하는 문제점 등이 있음

- 버전관리 시스템 (VCS) : 파일 및 변경이력등을 서버로 옮긴 것이다. 중앙의 서버가 파일과 변경이력을 관리하고 
		        변경이력이 있는 파일만 저장하여 효율적인 작업이 가능

2. git 사용법
 -  git bash 사용 (mingw64는 애뮬레이터의 한종류 윈도우즈 환경에서 gcc를 사용하게 해주는 프로그램)

 - git status(변경사항 조회) > git add(깃에 올릴 파일 추가) > git commit(git에 저장 commit의 상태로 저장하기에 코드의 수정이 용이하다)
  > git push(git에 올린다)
